# Antigravity Developer Instructions

## Mandatory Workflow Rules

### 1. Commit Permission
- **NEVER** use `git commit` or any git command that modifies the repository history automatically.
- **ALWAYS** implement the feature or fix first.
- **ALWAYS** present the results/verification to the user.
- **ALWAYS** explicitly ask for permission to commit. 
- **ONLY** proceed with a commit if the user provides explicit verbal or written approval (e.g., "yes", "commit", "proceed with commit").

### 2. UI Standards
- Prioritize professional, "Aero-Glass" and premium aesthetics.
- Avoid bulky buttons; use subtle integrated controls (Segmented Controls, Horizontal Chips).
- Maintain high-quality backdrop blurs and typography.

### 3. File Tracking
- Maintain `CHANGELOG.md` and `FEATURE_REGISTRY.md` locally.
- Keep `task.md` updated in the brain directory.
